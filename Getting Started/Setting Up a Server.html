Setting Up a Server

In order to learn Mythic, you will need a server to test everything on. This page will teach you how to set up a server that is hosted on your own computer.

localhost servers​

When you connect to a server, you join using an IP address, a unique identifier for every computer on the internet. localhost is simply a special IP that refers to your own computer. In Minecraft, you can connect to a server by typing in localhost as the server address to connect to your own server.

Localhost servers are great for testing things out but are unsuitable for public use. To make your server public, you will need to portforward. Portforwarding is not covered in this guide, but here is another guide that you can follow.

Setting up a server​

Downloading Java​

In order to run a Minecraft server, you will need to download Java. Most machines already have Java installed. You can check if you have Java installed by running:

java -version

If you see something like this:

java version "17.0.2" 2022-01-18 LTS
Java(TM) SE Runtime Environment (build 17.0.2+8-LTS-86)
Java HotSpot(TM) 64-Bit Server VM (build 17.0.2+8-LTS-86, mixed mode, sharing)
caution

Make sure that the version of Java you have installed is Java 17. Don't use newer versions of Java.

Then you have Java installed. If you see something like this:

bash: java: command not found

Then you do not have Java installed. PaperMC has a guide on how to do this.

Setting up a folder​

Create a folder somewhere on your computer (preferably not the Downloads folder as some machines automatically delete files in the Downloads folder after a while) and name it whatever you want. This folder will contain all the files for your server.

Downloading a server jar​

Server jars are the files that contain the internal code for the server. You can download the latest PaperMC server jar here.

Other server jars

Many other server jars exist, such as Spigot, CraftBukkit, Sponge, etc. However, most of them do not function well.

Lower forks of PaperMC, such as Spigot, lack the functionality and performance of PaperMC which Mythic relies on, and Sponge is a completely different system that is not compatible with Mythic.

Higher forks of PaperMC, such as Purpur and Airplane, work in most cases, but no guarantees can be made. If you encounter any issues, please replicate them on PaperMC before reporting them.

Cross-forks that enable compatibility with Mods, such as Mohist, fundamentally change the way the server works and are not compatible with most plugins. You might be able to get Mythic to work on these forks but no support will be provided.

Place the server jar in the folder you created earlier, and rename it to server.jar for convenience.

Creating a server launch script​

To launch your server, you will have to type in a command into the terminal. However, typing in the command every time you want to launch the server is tedious. Instead, you can create a script that will run the command for you.

Save this as start.sh. Next time, you can run it by typing sh ./start.sh in the terminal.

EULA​

Double click on the script you created earlier to launch the server. The server should start up and create a bunch of files in the folder you created earlier:

The one you want to look out for is eula.txt.

The Minecraft EULA is a legal document that you must agree to before you can run a Minecraft server. To agree to the EULA, open the eula.txt file and change eula=false to eula=true.

Now that you have agreed to the EULA, you can run the server again. If everything went well, you should see this message in the terminal:

[Server thread/INFO]: Done (x.xxxs)! For help, type "help"

Furthermore, there should be more files in the plugins folder:

You can ignore most of these files for now, but the ones you should pay attention to are:

  • server.properties - This file contains a lot of important settings for your server.
  • plugins - This folder contains all the plugins on your server.

Congratulations! You have successfully set up a server! This terminal window is now your server console, which you can use to run commands and view logs.

Plugins​

As explained in Background Information, plugins are things that change how the server works. Plugins are usually distributed as .jar files, which you can download from the internet. To install a plugin, simply place the .jar file in the plugins folder in your server folder.

In this case, we will install Mythic.

Note that every time you install or uninstall a plugin, you will have to restart the server. So restart your server again, and this time, connect to it using your Minecraft client.

Giving yourself OP​

OP is a special permission that allows you to do anything on the server. It's handy for testing things out, but you should never give OP to anyone else. Given that this is a localhost test server, it's fine to give yourself OP so you don't have to manually manage your permissions.

To give yourself OP on your server, go back to the console and type in:

op <your username>

Replace <your username> with your Minecraft username. You can now use any command on the server.

Conclusion​

Congratulations! You have successfully set up a local Minecraft testing server, and you have installed Mythic on it. You can now start learning how to use Mythic and create your first mob.
